## 📌 서비스 소개

**머문(Meomun)**은  
현재 내가 있는 **공간**에 머물렀던 감정, 생각, 짧은 한 줄을  
가볍게 남기고, 또 우연히 발견할 수 있는 **공간 기반 휘발성 메시지 앱**입니다.

- 💬 **대화/팔로우/피드 없이**  
  관계 맺기 부담 없이, 혼자 조용히 남기는 기록
- 📍 **공간에 묶인 메시지**  
  글은 타임라인이 아니라, **공간(위치)에 귀속**됩니다.
- 🕊 **휘발되는 메시지**  
  메시지는 **영구 보관되지 않고**, 일정 시간이 지나면 사라집니다.
- 👀 **나만 보는 것도, 남이 남긴 것도**  
  내가 남긴 메시지만 볼 수도 있고,  
  같은 공간을 지나간 누군가의 메시지를 우연히 발견할 수도 있습니다.

## ✨ 주요 기능

- **현재 위치 기반 메시지 작성**
  - 지금 내가 있는 공간을 기반으로 한 메시지 작성
  - 공간/위치 태그 자동 붙임 (예: 카페, 거리, 공원 등)

- **공간 위 메시지 버블**
  - 2D 혹은 3D 공간 위에 떠 있는 **메시지 버블 UI**
  - 최근 메시지 / 과거 메시지에 따라 다른 스타일 표시

- **휘발성 메시지**
  - 일정 시간이 지나면 자동으로 사라지는 메시지
  - "영원이 아닌 지금"만을 기록하는 경험

- **공간 기반 피드**
  - 지도나 추상화된 공간 뷰에서 주변 메시지 탐색
  - 내가 머문 장소들에 남긴 메시지를 다시 돌아볼 수 있는 가벼운 히스토리

## 🛠 기술 스택

- **Client**
  - iOS (Swift, SwiftUI)
  - MVI 기반 상태 관리 & 화면 전환 구조 실험
  - 위치 정보(위도/경도) 기반 UI 렌더링

- **Design**
  - Figma
  - 공간감을 느낄 수 있는 **3D/반추상 배경**
  - 말풍선(Message Bubble) 중심의 간단한 UI

## 🧱 아키텍처

- **Presentation (iOS)**
  - SwiftUI 기반 View
  - MVI 패턴으로 Intent → State → Render 흐름 구성

- **Domain**
  - `Message` 엔티티: 내용, 위치 정보, 생성/만료 시간 등
  - `Place` / `Location` 엔티티
  - 공간/시간 기준으로 메시지를 조회하는 유즈케이스

- **Data (진행 중)**
  - REST API / WebSocket 등을 통한 서버 연동 고려
  - 위치 기반 조회/등록 API
